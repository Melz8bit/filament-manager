## Work Done

### Raspberry Pi Deployment (Docker)

Deployed FilamentTracker to a Raspberry Pi that also runs OpenMediaVault (NAS) and Plex.

- Fixed `requirements.txt`, which was UTF-16LE-encoded (from a PowerShell `pip freeze >` redirect) — pip couldn't parse it, which would have silently broken `docker build`. Rewritten as plain UTF-8.
- Added `FORCE_HTTPS` setting (`config/settings.py`), decoupled from `DEBUG` — previously `SECURE_SSL_REDIRECT`/HSTS/secure-cookies were hardcoded on whenever `DEBUG=False`, which would force-redirect plain LAN `http://` access into a broken loop once the app was also reachable over HTTPS via a tunnel.
- Widened `.gitignore` to `.env*` (keeping `.env.example` tracked) so `.env.production` can never be committed by accident.
- Installed Docker on the Pi — hit an apt conflict where `get-docker.sh`'s generated `docker.list` and OMV's own `omvdocker.sources` both defined the same repo with different `Signed-By` keyrings. Resolved by removing the redundant `docker.list` and installing via apt using OMV's existing source.
- Created `.env.production` on the Pi (gitignored, recreated from `.env.example`'s shape) with `DEBUG=False`, `FORCE_HTTPS=False`, and the Supabase `DATABASE_URL`.
- `docker compose up --build -d` — confirmed migrations run and gunicorn serves cleanly on port 8000.

**Files changed:** `requirements.txt`, `config/settings.py`, `.env.example`, `.gitignore`

### Public Access: Migrated ngrok → Cloudflare Tunnel

Originally used ngrok (`barrier-rising-twine.ngrok-free.dev`) for HTTPS access (needed for browser camera/barcode-scanning, which requires a secure context). Free ngrok only reserves one hostname, which doesn't scale as more webapps get added to this Pi — migrated to a purchased domain instead.

- Registered `8bitcode.net` directly through Cloudflare (no separate nameserver step needed).
- Installed `cloudflared` on the Pi, authenticated, created tunnel `homepi` (ID `cc56089d-7538-4d11-b33c-c04e4e7f6de2`).
- Config at `/etc/cloudflared/config.yml` (had to be under `/etc/cloudflared`, not `~pi/.cloudflared` — `sudo cloudflared service install` resolves `~` to `/root`, not the invoking user's home).
- Ingress: `filament.8bitcode.net` → `http://localhost:8000`.
- Installed as systemd service (`cloudflared`), enabled, survives reboot.
- Confirmed barcode/camera scanning works over the tunnel's HTTPS.
- Fully decommissioned ngrok: stopped/uninstalled the service, removed the package, apt source, keyring, and config directory; removed ngrok references from `.env.production`.

### Mobile UI Fixes

- **Inventory**: top bar was crowded/overlapping on mobile (Product Library link + Cards/Grouped toggle + Add Spool button all in one row). Now stacks (title block, then actions row) below the `sm` breakpoint; the view-mode toggle is hidden on mobile entirely, and a fresh mobile session (detected via User-Agent sniff for "Mobi") now defaults to grouped view instead of cards.
- **Log a Print**: "No file? Enter manually →" link was wrapping into the Cancel/Continue buttons on narrow screens. Moved to its own line above them.
- **Print History**: table was forcing horizontal page scroll on mobile. Added a mobile-only (`sm:hidden`) card layout — print name (larger font) and color swatches share the top line, date/grams/cost below in small gray text — while the desktop table (now `hidden sm:block`) keeps its own internal `overflow-x-auto` scroll, unaffected.
  - Follow-up fix: the mobile card's name text had no wrap guard, so a long unspaced name (e.g. a raw filename with underscores) could overflow the card and drag the whole page into horizontal scroll. Added `break-words min-w-0` to fix.
- **Print names**: previously unbounded — file uploads fell back to the raw filename, failed URL-title fetches fell back to the raw URL, both of which could be long and messy. Added `PRINT_NAME_MAX_LENGTH = 60` (`tracker/constants.py`), applied via truncation at every name-setting call site (`LogPrintView` file/URL paths, `QueueAddView`, `QueueEditView`, `ManualEntryView`, `QueueFetchTitleView`), plus matching `maxlength="60"` on all four name `<input>`s. Manual entry already required a non-blank name (unchanged) — the fix targets the *silent-fallback-to-junk* paths, not that one.

**Files changed:** `templates/tracker/{spool_list,log_print,manual_entry,print_history,queue_add,queue_edit}.html`, `tracker/constants.py`, `tracker/views.py`

Note: no browser tool is available in this environment — changes were smoke-tested via Django test client (all pages return 200, mobile-UA default-view logic verified directly) and `manage.py check`, but not visually confirmed by Claude. User confirmed via screenshot after the fact and caught the card-overflow bug, which is now fixed.
