## Work Done

### Split Spool Slot: Inventory Dropdowns

Replaced raw color pickers in the split-slot form with spool inventory dropdowns.

**Why:** When splitting a slot (e.g. 131g of brown across two plates with different colors), the user should select actual spools from their inventory rather than picking an arbitrary hex color. This pre-assigns the spool FK on the new `PrintSpool` records so they arrive pre-selected on the assignment screen.

**Changes to `SplitSpoolSlotView` (`tracker/views.py`):**
- `get()` now passes `all_spools = Spool.objects.order_by("brand", "color_name", "material")` to the template
- `post()` reads `split_spool` (list of PKs) instead of `split_hex`; looks up each `Spool`, uses `spool.color_hex` as `slicer_hex`, and sets `spool` FK on newly created `PrintSpool` records
- `_build_slot()` checks `ps.spool_id` first — if pre-assigned during split, that spool stays selected on the assignment screen instead of falling back to color proximity matching

**New template (`templates/tracker/partials/split_slot_form.html`):**
- Each sub-slot row has a `<select name="split_spool">` listing all spools (brand, color name, material, grams remaining)
- A small color swatch next to each row updates live on `change` by reading `data-hex` from the selected option
- Used an HTML `<template>` element (rendered once by Django with all options) that JS clones for each row — no server round-trips for "Add sub-slot"
- IIFE initializes 2 default rows on load; exposes `splitAddRow` globally for the `+ Add sub-slot` button
