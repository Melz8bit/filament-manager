### Spool Run-Out Handling (Continuation Spool)

When a print slot requires more filament than the selected spool has left, the assignment screen now detects this automatically and prompts for a replacement spool.

**How it works:**
- Each spool `<option>` now carries `data-remaining` attribute
- On spool select change, JS `checkSpoolCapacity()` compares `remaining_g` to `grams_used`
- If insufficient: amber warning appears ("X.Xg left — Y.Yg will come from a replacement spool") and a second spool dropdown is shown
- When replacement is selected, a breakdown is shown: "X.Xg from first spool, then Y.Yg from replacement"
- Pre-selected spools are checked on page load — warning shows immediately if default match is short
- `SpoolAssignmentView.post()` handles `continuation_spool_<pk>` POST field:
  - Primary spool is decremented to 0 (uses all remaining)
  - New `PrintSpool` record created for continuation spool with the shortfall grams
  - Both spools decremented atomically inside `transaction.atomic()`

**Files changed:** `tracker/views.py`, `templates/tracker/partials/spool_slot_row.html`, `templates/tracker/spool_assignment.html`

### Depleted Spool Visibility Rules

- **Inventory list**: 0g spools hidden when a live sibling (same brand + color_name + material, `remaining_g > 0`) exists. Uses `Exists(OuterRef)` correlated subquery — no N+1.
- **Low-stock filter**: spools with `remaining_g < 100g` excluded from low-stock results when a well-stocked sibling (`remaining_g >= 100g`) exists.
- **Dashboard**: same well-stocked sibling exclusion applied to `low_stock_spools` query.

Rationale: if you have a fresh 1000g spool of the same color, the 0g empty or 20g nearly-empty spool shouldn't clutter the view or trigger warnings.

**Files changed:** `tracker/views.py` (`SpoolListView.get_queryset`, `dashboard` view)

### Grouped Inventory View

New "Grouped" view mode on the inventory page that collapses all spools of the same brand + color + material into a single card.

**Layout:** Card header with color swatch, brand, material badge. If >1 spool in group, shows total grams and spool count. Each spool gets its own row: progress bar, remaining/full grams, Low badge if applicable, Edit link and Delete button.

**Toggle:** "Cards / Grouped" segmented button in the inventory header. Preference saved to Django session — persists as you apply filters and navigate. Switching preserves all active filter params via `URL.searchParams.set('view', mode)`.

**Implementation:** `SpoolListView.get()` saves `?view=` to session. `get_context_data()` groups the already-fetched queryset using `itertools.groupby` (sorted Python-side by brand/color/material/-remaining_g). New `spool_group_card.html` partial renders the grouped card.

**Files changed:** `tracker/views.py`, `templates/tracker/spool_list.html`, `templates/tracker/partials/spool_group_card.html` (new)
